vCard Gateway

Introduction:

This page is part of an academic project developed within the Systems Integration course unit, taught by Professor Marisa Maximiano.
This project consists of developing a set of Rest API services with the objective of creating and supporting banking entities.

What is the project about:

The main aim of this project is to develop a Web API and an asynchronous notification system to support vCards (virtual
debit card associated to one mobile phone) representing the backend of the overall platform. To achieve that, the Web
API must be “connected” to one or more financial agencies (e.g., Bank1, Bank2, etc.) which means that banks are always
behind this Web API (no bank can be accessed directly) and user must have accounts on the available banks.
Another essential element of this project proposal is the administration console (Windows Form Based Application) which
could be used to register/make available banks in the Web API apart from monitoring the state of the system (if
something is down, etc.).
The full backend project implementation must be done using the technologies used in the practical classes of the Systems
Integration course (e.g., .NET, etc.). Also, all the developed RESTful services must have proper documentation to be
accessed and used by other developers. Also, all the features and implementation decisions should be explained in the
written report that needs to be delivered with the source code of the project.
